Paramètres de connexion de profil FTP

Profil FTP > Connexion

Ces paramètres contrôlent comment la connexion vers le serveur distant est réalisée.

Connexions simultanées

Spécifiez le nombre maximum de connexions (de 1 à 10) que Beyond Compare peut créer pour ce serveur.

Délai d'expiration de lecture (secondes)

Spécifiez le délai pour télécharger des fichiers.  L'opération échouera si aucun octet n'est reçu endéans ce temps.

Transferts passifs (PASV)

Certaines configurations de pare-feu requièrent ce paramètre qui modifie les méthodes utilisées pour transférer les données échangées entre votre ordinateur et le serveur FTP.  Tous les serveurs FTP ne supportent pas le mode PASV.  Si la connexion est suspendue, essayez de le cocher.

Limiter les ports utilisés pour des transferts actifs (PORT)

Spécifier un intervalle de ports auxquels le serveur FTP se connectera pour les téléchargements (envois/réceptions) en utilisant des transferts actifs (La case Transferts passifs est décochée).  Utilisez ceci si vous avez besoin de configurer votre pare-feu pour autoriser les connexions entrantes.

Utiliser HOST avant la connexion

Contrôle si la commande HOST doit être envoyée avant d'envoyer le nom d'utilisateur et le mot de passe pour se connecter.  Ceci est requis pour se connecter à certains hôtes FTP virtuels.  Désactivez-le si la connexion est perdue avant que le nom d'utilisateur ne soit envoyé.

Effacer le canal de contrôle (SSL uniquement)

Après l'envoi des détails encryptés de connexion, la communication du canal de contrôle est décryptée.  Essayez de l'enclencher si la connexion SSL initiale réussit mais que le transfert de fichier ou la liste de répertoire sont suspendus.

Effacer le canal de données (SSL uniquement)

Envoyer le contenu de fichier et les listes de dossiers non chiffrés.  Ceci peut apporter une accélération lors des transferts d'un grand nombre de fichiers après avoir envoyé les détails de connexion encryptés.

Note  Si vous avez besoin d'aide pour les paramètres de cette page, consultez votre administrateur système.
