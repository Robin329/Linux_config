Paramètres de conversion de Comparaison de Tableaux

Format tableau > Conversion

L'onglet Conversion définit comment les fichiers sont chargés et enregistrés.

Des sessions de Comparaison de Tableaux peuvent comparer directement des fichiers en texte brut, tels que les fichiers CSV (Valeurs Séparées par des Virgules).  Pour comparer des fichiers avec des formats structurés ou propriétaires, il n'est pas nécessaire d'extraire en premier le contenu du texte brut du fichier.  Par exemple, les documents Microsoft Word sont des fichiers binaires, mais Beyond Compare peut extraire et comparer leur contenu textuel.  Beyond Compare peut mettre en surbrillance les différences en termes de mots et de phrases, mais ne peut pas identifier des différences de formatage ou d'images imbriquées.

Sélectionnez une méthode de Conversion.  Lors de l'utilisation de méthodes Programme externe vous pouvez spécifier un programme de conversion avec le chemin d'accès approprié et les commutateurs de ligne de commande dans le champ Chargement.  Afin de supporter les noms de fichier avec des caractères étendus, soyez certain de bien choisir l'encodage du nom de fichier (Unicode ou ANSI).  Les variables suivantes seront remplacées par les informations indiquées :

 %s        fichier source

 %t        fichier cible

 %o        fichier original

On considère que les programmes de conversion se terminent avec succès s'ils retournent un code de sortie égal à zéro, et s'ils génèrent un fichier non vide.

Si vous avez un programme de conversion que vous voulez utiliser avant l'enregistrement, vous pouvez décochez Désactiver les modifications et dans le champ Enregistrement, spécifiez le chemin d'accès de ce programme de conversion et des commutateurs de ligne de commande.  Les variables ci-dessus s'appliquent.

L'Encodage de la plupart des fichiers texte peut être détecté automatiquement, mais vous pouvez spécifier quel page de code spécifique utiliser.

Cochez Ctrl+Z indique la fin de fichier si votre type de fichier utilise la valeur hexadécimale 1A comme un marqueur de fin de fichier (EOF, End Of File).
