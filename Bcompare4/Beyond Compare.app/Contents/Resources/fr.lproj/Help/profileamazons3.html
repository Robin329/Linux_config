Paramètres de profil Amazon S3

Profil S3 d'Amazon > Général

Type d'authentification

Contrôle d'où viennent les informations d'identification.

Informations d'identification S3 sauvegardées

Utilisez l'ID de la clé d'accès et la Clé d'accès secrète sauvegardés.

Charge à partir du fichier des informations d'identification

Charge la clé d'accès et la clé d'accès secrète à partir des informations d'identification AWS SDK ou d'un fichier de configuration AWS CLI.  Si saisies, les "tokens" de sécurité seront utilisées comme informations d'identification limitées dans le temps.  Ce fichier d'informations d'identification est %UserProfile%\.aws\credentials ou ~/.aws/credentials , mais Beyond Compare utilisera le fichier de "configuration" AWS CLI si nécessaire et qui respecte les variables d'environnemnt AWS_SHARED_CREDENTIALS_FILE et AWS_CONFIG_FILE si elles sont présentes.  Le profil doit exister dans ce fichier.  Si laissé à blanc, "default" sera utilisé.

Charge à partir des variables d'environnement

Charge la clé d'accès, la clé d'accès secrète et la "token" de sécurité à partir des variables d'environnement AWS_*.  Il testera d'abord les variables AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY et AWS_SESSION_TOKEN.  S'il n'existe pas, un profil sera sélectionné à partir du fichier des informations d'identification par défaut (comme ci-dessus).  La variable AWS_PROFILE peut être utilisée pour sélectionner un profil qui n'est pas par défaut.

Bucket (optionnel)

Ceci peut être utilisé pour sélectionner un "bucket" dans le cas où vous n'avez pas l'autorisation de lister tous les "buckets".

La Description peut inclure n'importe quel texte que vous souhaitez.
