Référence de script

Un script est un fichier texte simple contenant une liste de commandes qui peuvent contrôler le programme et exécuter automatiquement des opérations sur des fichiers ou produire des rapports.  Voir également Scripts et Exemples de scripts.

Notation

Signification

|

délimite les choix possibles

()

entoure une expression obligatoire

[]

entoure une expression facultative

<>

entoure la description d'un texte utilisateur obligatoire

[...]

suit une expression qui peut être répétée

Vous pouvez utiliser lt à la place de left (gauche) ou rt à la place de right (droite) dans n'importe quelle commande.

Voici les commandes de script (en ordre alphabétique) :

ATTRIB BEEP COLLAPSE COMPARE COPY COPYTO CRITERIA DATA-REPORT DELETE EXPAND FILE-REPORT FILTER FOLDER-REPORT HEX-REPORT LOAD LOG MOVE MOVETO MP3-REPORT OPTION PICTURE-REPORT REGISTRY-REPORT RENAME SELECT SNAPSHOT SYNC TEXT-REPORT TOUCH VERSION-REPORT

ATTRIB

Utilisation :

attrib (+|-)<ensemble d'attributs 1> [(+|-)<ensemble d'attributs 2> [...]]

 

Où <ensemble d'attributs> est :

[a][s][h][r]

 

Les versions Mac et Linux ne supportent pas attrib.

Définit (+) ou efface (-) les attributs DOS de fichier dans la sélection actuelle.  Utilisez a pour archive, s pour système, h pour caché (hidden) et r pour lecture seule (read-only).

Exemples :

attrib +h

attrib +sh -a

 

BEEP

Utilisation :

beep

 

Émet un bip dans le haut-parleur du PC.  Ce bip survient sans prendre en compte le commutateur de ligne de commande /silent.

COLLAPSE

Utilisation :

collapse all

collapse <chemin 1> [<chemin 2> [...]]

 

Réduit tous ("all") les dossiers ou des dossiers avec des chemins spécifiques.  Les chemins sont relatifs au dossier de base et ne peuvent pas contenir de caractères génériques.

Exemple :

collapse "Mon Dossier 1" "Mon Dossier 2"

 

COMPARE

Utilisation :

compare [CRC|binary|rules-based]

 

Compare le contenu de la sélection actuelle en utilisant le type de comparaison.  Par défaut, le type de comparaison le plus récent est utilisé.

Cette commande compare une seule fois les fichiers.  Pour modifier comment les fichiers sont comparés après une modification, utilisez CRITERIA.

COPY

Utilisation :

copy (left->right|right->left)

 

Copie les fichiers/dossiers sélectionnés dans la direction indiquée (left->right : de gauche à droite, right->left : de droite à gauche).

COPYTO

Utilisation :

copyto [left|right|all] [path:(relative|base|none)] <chemin>

 

Copie les fichiers/dossiers sélectionnés vers le côté spécifié (left = gauche, right = droite, all = tous) en utilisant l'option de chemin ("path").

•base conserve la structure du dossier complet, structure relative aux dossiers de base de tous les éléments sélectionnés.

•relative conserve la structure relative minimale de dossier de tous les éléments sélectionnés.

•Les valeurs par défaut sont all et path:none (aucune structure n'est conservée).

 

CRITERIA

Utilisation :

criteria [attrib:<ensemble d'attributs>] [version] [timestamp[:[<nombre de secondes>sec][;IgnoreDST]]] [size|CRC|binary|rules-based] [timezone:(ignore|(left|right)(+|-)<0..12>)] [follow-symlinks] [ignore-unimportant]

 

Où <ensemble d'attributs> est :

[a][s][h][r]

 

Les versions Mac et Linux ne supportent pas attrib.

Définit tous les paramètres de critères de comparaison.

•attrib autorise les comparaisons des attributs DOS de fichier qui sont spécifiés.  Utilisez a pour archive, s pour système, h pour caché (hidden) et r pour lecture seule (read-only).

•version autorise les comparaisons des informations de version usuelles dans les exe et dll.

•timestamp autorise les comparaisons d'horodatage.

o<nombre de secondes>sec considère des différences avec une telle tolérance de secondes comme des correspondances.

oIgnoreDST (ignorer l'heure d'été) considère les différences d'exactement 1 heure comme des correspondances.

•size (taille), CRC, binary (binaire) et rules-based (basée sur des règles) sont les manières possibles pour comparer les contenus.

•timezone (fuseau horaire) ignore les différences de fuseaux horaires et applique le décalage horaire au côté spécifié (left = gauche, right = droite).

•follow-symlinks (suivre les liens symboliques) traite les liens symboliques et les jonctions NTFS comme le fichier ou le dossier cible, au lieu du lien lui-même.

•ignore-unimportant traitent les différences non importantes comme des correspondances.

Exemple :

criteria attrib:sh timestamp:2sec;IgnoreDST rules-based timezone:left+6

 

DATA-REPORT

Utilisation :

data-report layout:<disposition> [options:<options>] [title:<titre du rapport>] output-to:(printer|clipboard|<nom de fichier>) [output-options:<options>] [<comparaison>]

 

Génère un rapport de Comparaison de Tableaux des fichiers sélectionnés actuellement.

•layout contrôle l'apparence de l'état, et peut valoir side-by-side, summary ou interleaved.

•options est facultatif et est utilisé différemment pour chaque disposition.

oignore-unimportant (ignorer les non-importants) traite les différences dans du texte non important comme des correspondances et est disponible pour toutes les dispositions.

odisplay-all (afficher tout), display-mismatches (afficher les différences) ou display-matches(afficher les correspondances) peut être utilisé pour contrôler quelles lignes de la comparaison seront incluses dans toutes les dispositions à l'exception de summary (résumé).  Par défaut, display-all (afficher tout) est utilisé.

oline-numbers inclut ce nombre de lignes dans la disposition side-by-side.

•Les arguments title (titre), output-to (sortir vers), output-options (options de sortie) et <comparaison> sont documentés sous Arguments communs de rapport.

DELETE

Utilisation :

delete [recyclebin=(yes|no)] (left|right|all)

 

Supprime les fichiers/dossiers actuellement sélectionnés dans le(s) côté(s) spécifié(s) (left = gauche, right = droite, all = tous).  L'argument recyclebin (corbeille) est effectif uniquement lorsque le système d'exploitation le supporte.  Les disques réseaux, les services distants et les fichiers archive ne le supportent pas.

EXPAND

Utilisation :

expand all

expand <chemin 1> [<chemin 2> [...]]

 

Développe les sous-dossiers spécifiés ou tous les sous-dossiers.  Les chemins sont relatifs au dossier de base et ne peuvent pas contenir de caractère générique.  Les sous-dossiers doivent être développés pour que leur contenu soit utilisable par des autres commandes telles que SELECT.  expand all (développer tout) ne développera pas de sous-dossiers qui ont été exclus de la comparaison par des paramètres de Filtre de fichier.

FILE-REPORT

Utilisation :

file-report layout:<disposition> [options:<options>] [title:<titre du rapport>] output-to:(printer|clipboard|<nom de fichier>) [output-options:<options>] [<comparaison>]

 

Génère un rapport des fichiers actuellement sélectionnés, rapport basé sur le type de fichiers traités.  Par exemple, cela produira un DATA-REPORT pour les types de fichier associés avec un format de Comparaison de Tableaux.

•layout (disposition) contrôle l'apparence des rapports et peut être side-by-side (côte à côte) ou summary (résumé).

•options est facultatif et est utilisé différemment pour chaque disposition.

oignore-unimportant (ignorer les non-importants) traite les différences dans du texte non important comme des correspondances et est disponible pour toutes les dispositions.

odisplay-all (afficher tout), display-mismatches (afficher les différences) ou display-matches(afficher les correspondances) peut être utilisé pour contrôler quelles lignes de la comparaison seront incluses dans toutes les dispositions à l'exception de summary (résumé).  Par défaut, display-all (afficher tout) est utilisé.

oline-numbers inclut les numéros de ligne dans la disposition side-by-side (côte à côte).

•Les arguments title (titre), output-to (sortir vers), output-options (options de sortie) et <comparaison> sont documentés sous Arguments communs de rapport.

Exemple :

file-report layout:side-by-side options:display-mismatches &

   output-to:printer output-options:print-color,wrap-word

 

FILTER

Utilisation :

filter <masques de fichier>

filter cutoff:([<|>](<horodatage>|<nombre de jours>[days])|none)

filter attrib:((+|-)<ensemble d'attributs>|none)

filter size:([<|>]<numéro>[KB|MB|GB|TB]|none)

filter exclude-protected

filter include-protected

 

Où <ensemble d'attributs> est :

[a][s][h][r]

 

Les versions Mac et Linux ne supportent pas attrib, exclude-protected, ni include-protected.

Contrôle quels dossiers et types de fichier sont comparés en utilisant le filtrage sur le nom (masques de fichier), filtrage de date (cutoff) ou un autre filtre (attrib, size, exclude-protected ou include-protected).

•Les filtres sur nom incluent et excluent les fichiers et dossiers basés sur leurs noms et chemins.  Des chemins multiples peuvent être séparés par des points-virgules.  Pour indiquer qu'un masque est un masque d'exclusion, utilisez un signe moins (-) devant.  Pour indiquer qu'un masque affecte des dossiers, utilisez un antislash (\) à la fin.

•cutoff (limite) exclut tous les fichiers plus anciens (<) ou plus récents (>) que l'horodatage ou un nombre de jours à partir de la date système actuelle (<nombre de jours>days).  Par défaut, cutoff:< est utilisé.  none (aucun) peut être utilisé pour enlever le filtrage de date.  L'analyse de date est assez indulgente, mais si vous voulez être sûr qu'elle analyse correctement, utilisez le format spécifié par les paramètres régionaux du système.  (Voir Panneau de configuration > Paramètres régionaux et linguistiques.)  L'heure est facultative et doit suivre la date (cutoff:"31/12/2012 14:00").  Souvenez-vous d'utiliser des guillemets (" ") lorsqu'un élément contient un espace.

•attrib limite la comparaison aux fichiers qui ont un attribut DOS de fichier inclus (+) et qui n'ont pas un attribut exclu (-).  Utilisez a pour archive, s pour système, h pour caché (hidden) et r pour lecture seule (read-only).  none (aucun) peut être utilisé pour enlever le filtrage d'un attribut.

•size (taille) exclut tous les fichiers plus petits (<) ou plus grands (>) que la taille donnée.  Par défaut, la taille est donnée en octets, mais vous pouvez ajouter Ko, Mo, Go ou To à la fin pour indiquer que la valeur est en kilooctet, mégaoctet, gigaoctet ou téraoctet respectivement.  none (aucun) peut être utilisé pour enlever le filtrage de taille.

•exclude-protected exclut les fichiers et dossiers qui ont les deux attributs système et caché positionnés.

•include-protected inclut les fichiers et dossiers qui ont les deux attributs système et caché positionnés.

Exemples :

filter "*.pas;*.dpr;*.dfm"

filter cutoff:<7days

filter cutoff:>"Dec 31, 2012"

filter cutoff:"12/31/2012 12:00pm"

filter attrib:-sh

filter "*.txt;-Mon Dossier\" attrib:+r

 

FOLDER-REPORT

Utilisation :

folder-report layout:<disposition> [options:<options>] [title:<titre du rapport>] output-to:(printer|clipboard|<nom de fichier>) [output-options:<options>]

 

Génère un rapport de Comparaison de Dossiers des dossiers de base actuellement chargés.  Les dossiers sont montrés dans leur état actuel.  Pour inclure les sous-dossiers, ajoutez une commande expand all (développer tout) avant celle-ci.

•layout contrôle l'apparence du rapport, et peut être side-by-side, summary ou xml.

•options est facultatif.

oLes indicateurs de filtres d'affichage contrôlent quels fichiers sont inclus dans le rapport.  Les valeurs possibles sont display-all (afficher tout), display-mismatches (afficher les différences), display-no-orphans (afficher les non-orphelins), display-mismatches-no-orphans (afficher les non-orphelins et les différents), display-orphans (afficher les orphelins), display-left-newer (afficher les plus récents à gauche), display-right-newer (afficher les plus récents à droite), display-left-newer-orphans (afficher les orphelins et les plus récents à gauche), display-right-newer-orphans (afficher les orphelins et les plus récents à droite) et display-matches (afficher les correspondances).  Par défaut, display-all (afficher tout) est utilisé.

oDes colonnes facultatives peuvent être incluses, séparées par des virgules, dans le rapport en utilisant les balises column-version, column-size (taille), column-crc, column-timestamp (horodatage), column-attributes, column-group, column-owner (propriétaire), et column-vcs.  Par défaut, column-size (taille) et column-timestamp (horodatage) sont incluses.  column-none (aucune) peut être utilisée par elle-seule pour enlever les colonnes par défaut.

oUtilisez l'indicateur include-file-links pour générer des rapports de fichiers et y inclure des liens s'y référant dans le rapport de dossiers.  Ce n'est qu'un rapport de dossiers qui est et côte à côte, et HTML peut utiliser cet indicateur.

•Les arguments title (titre), output-to (sortir vers) et output-options (options de sortie) sont documentés sous Arguments communs de rapport.

Exemples :

folder-report layout:side-by-side &

   options:column-size,column-version,column-timestamp &

   output-to:clipboard output-options:html-color

folder-report layout:xml output-to:"C:\Mon Rapport.xml"

LOAD

Utilisation :

load <session>

load [create:(all|left|right)] <chemin gauche> [<chemin de droite>]

load <default>

 

Charge la session de Comparaison de Dossiers, les dossiers de base spécifiés, ou une nouvelle Comparaison de Dossiers avec les paramètres par défaut ( load <default>).  Un échec du chargement d'un dossier de base provoquera l'arrêt du script.

•Le chargement d'une session entraîne l'utilisation de tous les paramètres de cette session, excepté le filtre d'affichage (afficher tout, uniquement les non-correspondances, seulement les correspondances, etc.).

•Le chargement des dossiers de base utilise les paramètres actuellement actifs.  Par défaut, les défauts du programme sont utilisés.

•Le paramètre facultatif create crée un dossier de base s'il n'existe pas déjà.

Exemples :

load "Ma Session"

load "C:\Mon Dossier" "X:\Votre dossier"

load <default>

 

LOG

Utilisation :

log [none|normal|verbose] [[append:]<nom de fichier>]

 

Contrôle le niveau de détail du journal et où est stocké le journal.

•none (aucun), normal et verbose (verbeux) détermine le niveau de détail utilisé dans le journal.  La journalisation est initialement désactivée et commence lorsque le mode est changé de none.

•append: causes la journalisation à s'ajouter à un fichier au lieu de le remplacer.  Par défaut, le journal est créé comme Log.txt dans le dossier courant (ex. le dossier Windows "Démarrer dans" dans les propriétés de raccourci).

•base conserve la structure du dossier complet, structure relative aux dossiers de base de tous les éléments sélectionnés.

•relative conserve la structure relative minimale de dossier de tous les éléments sélectionnés.

•Les valeurs par défaut sont all et path:none (aucune structure n'est conservée).

OPTION

Utilisation :

option stop-on-error

option confirm:(prompt|yes-to-all|no-to-all)

 

Ajuste les options de traitement du script.

•stop-on-error (stopper en cas d'erreur) fait que le script attend diverses conditions d'erreur, incluant les erreurs d'opérations sur des fichiers, et, lorsqu'une survient, demande à l'utilisateur avant de continuer.

•confirm (confirmer) peut utiliser prompt (demander), yes-to-all (oui à tout) ou no-to-all (non à tout) pour traiter les dialogues de confirmation qui surviennent suite aux opérations sur des fichiers.  Par défaut, prompt est utilisé.

RENAME

Utilisation :

rename [regexpr <ancien masque>] <nouveau masque>

 

Exécute un renommage multi-fichier sur la sélection active en utilisant le masque nouveau spécifié.  regexpr utilise un masque ancien pour exécuter un renommage via expressions régulières.  Par défaut, un renommage du style DOS est exécuté.

Exemples :

rename *.bak

rename regexpr (...)(...)\.txt $2$1.txt

 

SELECT

Utilisation :

select <masque de sélection 1> [<masque de sélection 2> [...]]

 

Où <masque de sélection> est un parmi :

all

[(left|right|all).][(exact|diff|newer|older|orphan|all).][(files|folders|all)]

empty.folders

 

Contrôle sur quels fichiers et dossiers on peut travailler.  Uniquement les fichiers et dossiers qui correspondent à au moins un des masques de sélection font partie de la sélection.

•left (gauche), right (droite) ou all (tous) limite la sélection au côté spécifié.  Par défaut, all (tous) est utilisé.

•exact (exact), diff (différent), newer (plus récent), older (plus ancien), orphan (orphelin) ou all (tous) limite la sélection en se basant sur les résultats de la comparaison.  diff se réfère aux différences pour des critères autres que l'horodatage.  Pour comparer les horodatages, utilisez newer (plus récent) ou older (plus ancien) à la place.  Les dossiers doivent contenir uniquement le type de résultat spécifié qui est concerné.  Par défaut, all (tous) est utilisé.

•files (fichiers), folders (dossiers) ou all (tous) limite la sélection aux fichiers, aux dossiers ou aux deux.  Par défaut, all (tous) est utilisé.

•empty.folders est un masque de sélection spéciale qui sélectionne les dossiers vides.

Exemples :

select all

select left

select exact

select files

select right.diff

select left.folders

select newer.files right.older.files

 

SNAPSHOT

Utilisation :

snapshot [save-crc] [save-version] [expand-archives] [follow-symlinks] [include-empty] [no-filters] left|right|path:<chemin> [output:<cible>]

 

Enregistre un Instantané en lecture seule, du dossier spécifié.

•save-crc (sauve les crc) calcule les CRC de tous les fichiers et les stocke dans l'Instantané.

•save-version (sauve la version) stocke l'information de version des fichiers exécutables.

•expand-archives (développe les archives) stocke le contenu des fichiers archives dans un format étendu.

•follow-symlinks (suit les liens symboliques) traite les fichiers et dossiers indiqués par les liens symboliques.

•include-empty stocke les dossiers qui sont vides ou qui contiennent des fichiers exclus.

•no-filters (pas de filtre) stocke tous les fichiers sans prendre en compte les filtres actifs de nom de fichier.

•left (gauche), right (droite) et path (chemin) indiquent quel dossier est stocké.  left et right enregistreront un Instantané d'un des dossiers de base chargé, et path enregistrera un Instantané du chemin spécifié.

•output (sortie) est facultatif et détermine où l'Instantané sera enregistré.  Par défaut, il sera sauvé dans le dossier courant.

oSi la cible est un nom de fichier pleinement qualifié, ce sera le nom de l'Instantané.  Si la cible n'a pas d'extension, .bcss sera utilisé.

oSi la cible n'est pas incluse ou se réfère à un dossier, un nom de fichier sera généré basé sur le nom du dossier source et la date système actuelle, ex. dossier_2012-12-31.bcss.  Si nécessaire, un numéro additionnel sera ajouté au nom généré pour l'empêcher d'écraser des Instantanés existants.

Exemples :

snapshot save-crc save-version no-filters path:C:\ output:D:\

snapshot left output:"Mon Instantané.bcss"

 

SYNC

Utilisation :

sync [visible] [create-empty] (update|mirror):(left->right|right->left|all)

 

Synchronise des dossiers en copiant et supprimant des fichiers.  Cette commande n'utilise pas la sélection active.

•visible autorise le comportement "Affecte uniquement les fichiers visibles".  Uniquement les fichiers dans des dossiers développés seront concernés.

•create-empty autorise le comportement "Créer les dossiers vides".

•update (mettre à jour) copie les fichiers plus récents et les orphelins dans la direction indiquée, laissant inchangés les orphelins et les plus récents de l'autre côté.  L'utilisation de all (tous) comme direction, indique une synchronisation bidirectionnelle, et copie les fichiers plus récents et les orphelins dans les deux directions.

•mirror (miroir) rend le dossier cible identique à celui de la source.  Les fichiers plus récents, plus anciens, différents et orphelins sont copiés tous vers la destination.  Les orphelins de la destination sont supprimés.  La direction all (tous) ne s'applique pas dans ce contexte, et n'est pas valide.

Exemple :

sync create-empty mirror:left->right

 

TEXT-REPORT

Utilisation :

text-report layout:<disposition> [options:<options>] [title:<titre du rapport>] output-to:(printer|clipboard|<nom de fichier>) [output-options:<options>] [<comparaison>]

 

Génère un rapport de Comparaison de Textes des fichiers sélectionnés actuellement.

•layout (disposition) contrôle l'apparence des rapports et peut être side-by-side (côte à côte), summary (résumé), interleaved (entrelacé), patch, statistics ou xml.  Les anciennes dispositions over-under (dessus dessous) et composite (composée) ont été remplacées par interleaved.

•options est facultatif et est utilisé différemment pour chaque disposition.

oignore-unimportant (ignorer les non-importants) traite les différences dans du texte non important comme des correspondances et est disponible pour toutes les dispositions.

odisplay-all (afficher tout), display-mismatches (afficher les différences), display-context (afficher le contexte) ou display-matches (afficher les correspondances) peuvent être utilisés pour contrôler quelles lignes de la comparaison seront incluses dans toutes les dispositions excepté summary (résumé), patch et statistics.  Par défaut, display-all (afficher tout) est utilisé.

oline-numbers inclut les numéros de ligne dans la disposition side-by-side (côte à côte).

ostrikeout-left-diff (barrer les lignes de différence à gauche) biffe les lignes qui sont seulement à gauche, dans la disposition interleaved (entrelacé).

ostrikeout-right-diff (barrer les lignes de différence à droite) biffe les lignes qui sont seulement à gauche, dans la disposition interleaved (entrelacé).

oLes formats patch-normal (patch normal), patch-context (patch contextuel) ou patch-unified (patch unifié) sont disponibles dans la disposition patch.  Par défaut, patch-normal est utilisé.

•Les arguments title (titre), output-to (sortir vers), output-options (options de sortie) et <comparaison> sont documentés sous Arguments communs de rapport.

L'ancienne option stats-descriptive générera un rapport résumé à la place après avoir généré un avertissement dans le journal.  L'ancienne option stats-tabular est ignorée dans la disposition statistics : la disposition statistics est maintenant toujours tabulaire (CSV).

Exemples :

text-report layout:interleaved options:display-context &

   output-to:printer output-options:print-color,wrap-word

text-report layout:patch options:patch-unified &

   output-to:"Mon Rapport.txt"

 

TOUCH

Utilisation :

touch (left->right|right->left)

touch (left|right|all):(now|<horodatage>)

 

Copie les horodatages des fichiers d'un côté vers l'autre (left = gauche, right = droite) ou définit les horodatages de tous ("all") les fichiers d'un côté spécifié à l'heure système actuelle (now) ou à l'horodatage spécifié.  Vous devez sélectionner les fichiers avant de pouvoir les horodater.

VERSION-REPORT

Utilisation :

version-report layout:<disposition> [options:<options>] [title:<titre du rapport>] output-to:(printer|clipboard|<nom de fichier>) [output-options:<options>] [<comparaison>]

 

Génère un rapport de Comparaison de Versions des fichiers sélectionnés actuellement.

•layout (disposition) contrôle l'apparence des rapports et peut être side-by-side (côte à côte) ou summary (résumé).

•options est facultatif et est utilisé différemment pour chaque disposition.

oignore-unimportant (ignorer les non-importants) traite les différences dans du texte non important comme des correspondances et est disponible pour toutes les dispositions.

odisplay-all (afficher tout), display-mismatches (afficher les différences) ou display-matches(afficher les correspondances) peut être utilisé pour contrôler quelles lignes de la comparaison seront incluses dans toutes les dispositions à l'exception de summary (résumé).  Par défaut, display-all (afficher tout) est utilisé.

•Les arguments title (titre), output-to (sortir vers), output-options (options de sortie) et <comparaison> sont documentés sous Arguments communs de rapport.

Arguments communs de rapport

Les commandes de rapport (data-report rapport Données, file-report rapport Fichiers, folder-report rapport Dossiers, hex-report rapport Hexadécimal, mp3-report rapport MP3, picture-report rapport Pixels, registry-report rapport Registre, text-reportrapport Texte et version-report rapport Version) doivent inclure une information sur l'endroit où le rapport doit être envoyé, et peut inclure une information additionnelle sur le format de sortie.

•title (titre) contrôle le titre qui apparaît en tête de rapport.

•output-to (sortie vers) contrôle la cible en sortie.  Cela peut être printer (imprimante), clipboard (presse-papiers) ou un nom de fichier.

•output-options (options de sortie) est facultatif et est utilisé différemment pour chaque cible en sortie.

oLes schémas de couleur print-color (impression en couleur) ou print-mono (impression en monochrome) sont disponibles pour une sortie printer (imprimante).  Par défaut, print-mono est utilisé.

oLes orientations print-portrait ou print-landscape (paysage) sont disponibles pour une sortie printer.  Par défaut print-portrait est utilisé.

owrap-none (pas de retour automatique à la ligne), wrap-character (retour sur base d'un caractère) ou wrap-word (retour sur base de mot) contrôlent quand les longues lignes sont coupées.  La sortie Printer (imprimante) peut utiliser toutes les trois options.  La sortie HTML peut utiliser wrap-none et wrap-word.  Par défaut, wrap-none est utilisé.

ohtml-color, html-mono ou html-custom est requis pour une sortie HTML au lieu de texte brut.  Ces options sont disponibles pour les sorties clipboard (presse-papiers) et file (fichier).  html-custom requiert le nom de fichier ou l'URL de la feuille de style externe.

•<comparaison> peut être soit un nom de session, soit une paire de noms de fichier.  Le rapport de fichier utilisera la comparaison spécifiée au lieu des fichiers qui ont été sélectionnés dans le script.  Lors de l'utilisation d'une session sauvegardée, le type de comparaison doit correspondre au type de rapport (ex. une session de Comparaison de Tableaux devra utiliser data-report ou file-report).

Exemples :

data-report layout:interleaved output-to:printer &

   output-options:print-color,print-landscape

file-report layout:summary output-to:clipboard &

   output-options:wrap-word,html-color

text-report layout:patch options:patch-unified &

   output-to:"Mon Rapport.txt"
