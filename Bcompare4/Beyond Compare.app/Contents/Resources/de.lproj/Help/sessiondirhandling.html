Einstellungen für Ordnervergleichsbehandlung

Sitzung > Sitzungseinstellungen > Behandlung

Ordnerbehandlung

Unterordner automatisch im Hintergrund durchlaufen bewirkt das Lesen der Ordner während der Vergleich geladen wird.  Auf diese Weise können die Ordner entsprechend gefärbt werden, bevor Sie sie öffnen.  Ordner-Singles oberster Ebene können gewöhnlicherweise ohne Durchlaufen gefärbt oder gefiltert werden, da sie nur weitere Singles enthalten können.  Unterordner von Ordner-Singles oberster Ebene durchlaufen bewirkt, dass Ordner-Singles auf jeden Fall durchlaufen werden, so dass die Ordnergröße exakt ermittelt werden kann.

Unterordner beim Laden der Sitzung öffnen öffnet sämtliche Ordner automatisch beim Laden des Vergleiches.  Nur Unterordner mit Unterschieden öffnen beschränkt das automatische Öffnen auf jene Ordner, in denen Unterschiede festgestellt werden.

Archivbehandlung (z.B. ZIP-Dateien)

Immer als Dateien behandelt Archivdateien wie andere Dateien.  Sie werden zusammen mit Dateien aufgelistet und sortiert, und erhalten kein Ordnersymbol.

Wenn geöffnet wie Ordner sortiert und vergleicht Archivdateien wie andere Dateien, jedoch mit einem ZIP-Ordnersymbol.  Wenn Sie den ZIP-Ordner einmal geöffnet haben, ändert sich sein Vergleichsstatus, um wie bei anderen Ordnern den Inhalt widerzuspiegeln.

Immer als Ordner behandelt Archivdateien wie reguläre Ordner.

Dateibehandlung

Zeitstempel lokaler Dateien beim Kopieren auf FTP-Standorte anpassen

Viele FTP-Standorte erlauben Beyond Compare nicht, die Zeitstempel von Dateien zu steuern.  Nach dem Kopieren einer Datei auf den FTP-Standort wird das Dateidatum also nicht zu jenem der lokalen Datei passen.  Diese Option ändert den Zeitstempel der lokalen Datei, so dass er mit jenem auf dem FTP-Standort übereinstimmt.

Symbolischer Verknüpfung folgen

Zeigt symbolische Verknüpfungen und sog. NTFS-Junction-Points an als wären sie ihr Ziel.  Dies beinhält, ob sie als Dateien oder Ordner angezeigt werden, ihre Größe, das Datum ihrer letzten Änderung und ihre Attribute.

DOS 8.3-Dateinamen beibehalten

Versucht dieselben kurzen (DOS 8.3) Dateinamen beim Kopieren von Dateien zu erhalten.

NTFS-Dateiberechtigungen kopieren

Kopiert Sicherheitsdeskriptoren von Dateien auf NTFS-Dateisystemen unter Windows 2000 oder neuer.  Diese Funktion behält keine Dateirechte von Dateien bei, die in Textvergleichssitzungen geändert wurden.

Erstellungsdatum kopieren

Behält das ursprüngliche Erstellungsdatum einer lokalen Datei bei, wenn eine Kopie erstellt wird.

Autom. Aktualisierung

Setzen Sie diese Option, um den Ordnervergleich regelmäßig aktualisieren zu lassen.
