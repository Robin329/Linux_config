Amazon S3 Profileinstellungen

Amazon S3 Profil > Allgemeines

Authentifizierungstyp

Steuert woher Credentials geladen werden.

Gespeicherte S3 Credentials

Gespeicherte Zugriffsschlüssel-ID und Geheimer Zugriffsschlüssel verwenden.

Aus der Credentials-Datei laden

Den Zugriffsschlüssel und geheimen Zugriffsschlüssel aus einer AWS SDK Credentials-Datei oder AWS CLI Config-Datei laden.  Sofern vorhanden, werden Security Tokens für zeitlich begrenzte Credentials verwendet.  Die Standard-Credentials-Datei ist %UserProfile%\.aws\credentials oder ~/.aws/credentials , aber Beyond Compare wird wenn nötig die AWS CLI "config" Datei verwenden, und die AWS_SHARED_CREDENTIALS_FILE und AWS_CONFIG_FILE Umgebungsvariablen berücksichtigen, sofern sie existieren.  Das Profil sollte eines sein, welches in der Datei existiert.  Wenn das Feld leer bleibt, wird "Standard" verwendet.

Aus den Umgebungsvariablen laden

Den Zugriffsschlüssel, geheimen Zugriffsschlüssel und das Security-Token aus AWS_* Umgebungsvariablen laden.  Zuerst wird nach den Variablen AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY sowie AWS_SESSION_TOKEN gesucht.  Wenn diese nicht vorhanden sind, wird ein Profil aus der Standard-Credentials-Datei (wie oben) ausgewählt.  Die AWS_PROFILE Variable kann zur Auswahl eines Nicht-Standard-Profils verwendet werden.

Bucket (optional)

Dies kann zur Auswahl eines Bucket für den Fall verwendet werden, dass Sie keine Berechtigung haben, alle Buckets aufzulisten.
