Einstellungen für Ordnersynchronisationsbehandlung

Sitzung > Sitzungseinstellungen > Behandlung

Ordnerbehandlung

Leere Ordner synchronisieren berücksichtigt leere Ordner oder Ordner, deren kompletter Inhalt herausgefiltert wurde.

Unterordner beim Laden der Sitzung öffnen öffnet sämtliche Ordner automatisch beim Laden des Vergleiches.

Innerhalb von Archiven synchronisieren behandelt Archivdateien wie reguläre Ordner.

Dateibehandlung

Löschen verschiebt in den Papierkorb, wenn möglich

Verschiebt Dateien und Ordner von lokalen Laufwerken in den Papierkorb, statt sie dauerhaft komplett zu löschen.  Diese Option wird beim Löschen von Netzlaufwerken, entfernten Diensten und innerhalb von Archivdateien ignoriert.

Zeitstempel lokaler Dateien beim Kopieren auf FTP-Standorte anpassen

Viele FTP-Standorte erlauben Beyond Compare nicht, die Zeitstempel von Dateien zu steuern.  Nach dem Kopieren einer Datei auf den FTP-Standort wird das Dateidatum also nicht zu jenem der lokalen Datei passen.  Diese Option ändert den Zeitstempel der lokalen Datei, so dass er mit jenem auf dem FTP-Standort übereinstimmt.

Symbolischer Verknüpfung folgen

Zeigt symbolische Verknüpfungen und sog. NTFS-Junction-Points an als wären sie ihr Ziel.  Dies beinhält, ob sie als Dateien oder Ordner angezeigt werden, ihre Größe, das Datum ihrer letzten Änderung und ihre Attribute.

DOS 8.3-Dateinamen beibehalten

Versucht dieselben kurzen (DOS 8.3) Dateinamen beim Kopieren von Dateien zu erhalten.

NTFS-Dateiberechtigungen kopieren

Kopiert Sicherheitsdeskriptoren von Dateien auf NTFS-Dateisystemen unter Windows 2000 oder neuer.  Diese Funktion behält keine Dateirechte von Dateien bei, die in Textvergleichssitzungen geändert wurden.

Erstellungsdatum kopieren

Behält das ursprüngliche Erstellungsdatum einer lokalen Datei bei, wenn eine Kopie erstellt wird.

Autom. Aktualisierung

Setzen Sie diese Option, um den Ordnervergleich regelmäßig aktualisieren zu lassen.
